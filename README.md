№1. Що таке immutable об’єкт? Для чого об’єкти класу String зробили імутабельними? Чому клас String задекларований як final?

Immutable (незмінний) об’єкт — це об’єкт, стан якого не можна змінити після створення.
Будь-яка “зміна” створює новий об’єкт.

Чому String immutable:

Безпека: рядки часто використовується в паролях, URL, конфігурації, ClassLoader — зміна могла б створити загрози.

Thread-safety: immutable об’єкти безпечні для роботи з потоками без синхронізації.

Кешування (String pool): JVM може кешувати рядки в String Pool. Якби вони були змінні, кешування було б небезпечним і неможливим.

Чому String final:
Щоби ніхто не міг успадкувати клас і змінити поведінку immutable-логіки.

№2. Що таке регулярні вирази? Наведіть приклади регулярних виразів.

Регулярні вирази (regex) — це шаблони для пошуку та обробки тексту.

Приклади:

\d+ — одна або більше цифр

[A-Za-z]+ — латинські літери

^\w+@\w+\.\w{2,3}$ — проста перевірка email

\s+ — один або більше пробілів

^[A-Z][a-z]+$ — слово, що починається з великої літери

№3. В чому полягає різниця між оператором «==» та методом «equals()»?

== перевіряє, чи два посилання вказують на той самий об’єкт у пам’яті.

equals() перевіряє, чи значення об’єктів однакові (залежить від реалізації методу).

Приклад:

String a = new String("hello");
String b = new String("hello");

System.out.println(a == b);      // false (різні об’єкти)
System.out.println(a.equals(b)); // true  (однаковий зміст)


---


№4. Для чого потрібні класи StringBuilder та StringBuffer?

Вони призначені для зміни рядків без створення нових об’єктів, на відміну від String.

StringBuilder — швидкий, НЕ потокобезпечний

StringBuffer — повільніший, але потокобезпечний (має synchronized методи)

Використовуються для частого конкатенування рядків (цикли, форматування).


---

№5. Яким чином простіше всього прибрати пробіли на початку та кінці об’єкту String?

Використати метод trim():

String text = "   Hello world   ";
System.out.println(text.trim()); // "Hello world"
